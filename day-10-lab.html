`count` Meta‑Argument

The simplest looping method is using count. It tells Terraform how many copies of a resource to create. 


resource "aws_instance" "web" {
  count         = 3
  ami           = "ami-12345678"
  instance_type = "t2.micro"
}

With count = 3, Terraform creates three instances automatically. [oai_citation:3‡DEV Community](https://dev.to/patdevops/mastering-loops-and-conditionals-in-terraform-5a16?utm_source=chatgpt.com)

`for_each` for Lists and Maps

For more control, especially with maps or lists, use for_each. This loops over each item and lets you reference both keys and values. 


variable "instances" {
  type = map(string)
  default = {
    "app1" = "t2.micro"
    "app2" = "t2.small"
  }
}

resource "aws_instance" "web" {
  for_each      = var.instances
  instance_type = each.value
  ami           = "ami-12345678"

  tags = {
    Name = each.key
  }
}

Each instance gets a unique name and type based on the map entries. [oai_citation:4‡DEV Community](https://dev.to/patdevops/mastering-loops-and-conditionals-in-terraform-5a16?utm_source=chatgpt.com)

`for` Expressions

Terraform also provides list and map comprehensions using for, letting you transform collections without resource blocks. 


locals {
  names = ["dev", "test", "prod"]
  tags  = [for n in local.names : "${n}-env"]
}

Using Conditionals

Conditionals let Terraform decide values based on a logical expression. The syntax is simple: 


condition ? true_value : false_value

Conditional Attributes

You can use conditionals anywhere Terraform accepts expressions. For example, selecting an instance type based on environment: 


resource "aws_instance" "example" {
  instance_type = var.is_prod ? "t3.large" : "t3.micro"
  ami           = "ami-12345678"
}

Conditional Resource Creation

Terraform doesn’t skip resources dynamically, but you can combine a conditional with count to control creation: 


resource "aws_instance" "optional" {
  count = var.enable_instance ? 1 : 0
  ami   = "ami-12345678"
}

Setting count = 0 means Terraform creates no instances of that resource. [oai_citation:5‡DEV Community](https://dev.to/patdevops/mastering-loops-and-conditionals-in-terraform-5a16?utm_source=chatgpt.com)
