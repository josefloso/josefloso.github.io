Terraform AWS Setup Guide

Terraform lets you define infrastructure as code. Instead of clicking around in AWS, you write configuration files and Terraform provisions everything for you.

Prerequisites

  • Terraform installed (terraform -version)
  • AWS account
  • IAM user with access keys

Step 1: Set up the AWS Provider

provider "aws" {
  region = "us-east-1"
}

Step 2: Configure AWS Credentials

export AWS_ACCESS_KEY_ID="your-access-key"
export AWS_SECRET_ACCESS_KEY="your-secret-key"
export AWS_DEFAULT_REGION="us-east-1"

Step 3: Initialize Terraform

terraform init

What it does:

  • Downloads required providers
  • Prepares the working directory

Step 4: Add a Resource (S3 Bucket)

resource "aws_s3_bucket" "day3_bucket" {
  bucket = "tf-day3-bucket-joseph"

  tags = {
    Name    = "day3-s3-bucket"
    Purpose = "Terraform Lab"
  }
}

Step 5: Plan the Deployment

terraform plan

What it does:

  • Shows what Terraform will create
  • Does not make changes

Step 6: Apply the Configuration

terraform apply

Creates the resources after confirmation.

Step 7: Clean Up

terraform destroy

This removes all created infrastructure.

Key Terraform Commands Summary

Command Purpose
terraform init Initialize project
terraform plan Preview changes
terraform apply Create/update infrastructure
terraform destroy Delete all resources

Conclusion

Terraform simplifies infrastructure by turning it into code. Once you understand providers, resources, and core commands, you can reliably deploy and manage infrastructure.
