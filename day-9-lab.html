Module Structure

The challenge focuses on building reusable Terraform modules with proper inputs, outputs, and calling configurations.

Goals

  • Understand the difference between root and child modules
  • Parameterize modules to make them reusable across environments
  • Use outputs to expose important information (e.g., ALB DNS, instance IDs)
  • Refactor existing Terraform code into proper module structure

Key Learnings

  • Child modules are like functions — reusable, encapsulated units of Terraform configuration.
  • Root modules orchestrate child modules and provide environment-specific variable values.
  • Using terraform init downloads and prepares module sources.
  • Outputs from child modules are captured in the state file and can be referenced by other modules or root configurations.

Calling the Module


module "webserver" {
  source        = "../modules/example-module"
  cluster_name  = "dev-cluster"
  instance_type = "t3.micro"
  min_size      = 1
  max_size      = 2
}

Outputs

  • ALB DNS name to access the cluster
  • Instance IDs and security group IDs for reference

Refactoring Observations

  • Added safe defaults for easier dev use
